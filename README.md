# SeMIS v2 — 에어제타 보안종합정보시스템

기존 [kjsemis Google Sites](https://sites.google.com/view/kjsemis/)를 고도화한 Version 2.
정적 웹앱(HTML/CSS/JS)으로 어떤 웹서버(GitHub Pages 등)에서도 바로 호스팅 가능.

**접속 주소: https://mark4mission.github.io/semis-v2/** (GitHub Pages, `main` 브랜치 push 시 자동 반영)

## 구조

```
SeMIS_v2/
├── index.html       # 앱 셸 (로그인 모달, 헤더, 사이드바, 메인)
├── css/main.css     # 디자인 시스템 (반응형)
├── js/
│   ├── app.js       # 코어 엔진: 인증(SHA-256), 저장소+마이그레이션, 메뉴 엔진, 권한, 라우터
│   ├── modules.js   # 모듈: 대시보드(공지/보안등급), 시스템설정
│   ├── calendar.js  # 모듈: 일정관리 캘린더 (v2.1)
│   └── sync.js      # Supabase 공용 DB 실시간 동기화 (v2.1)
└── tests/
    └── run-tests.cjs # jsdom 테스트 스위트 (116건, CommonJS)
```

## 핵심 기능

- **로그인**: 모달에서 암호만 입력하는 간편 로그인. 암호는 SHA-256 해시로만 코드에 보관 (평문 미노출)
- **권한 3단계**: 시스템관리자(`mark3464`) / 보안관리자(`avsec`) / 일반사용자(`branch`)
- **메뉴 엔진**: 기존 사이트 전체 메뉴를 외부링크 메뉴로 이관 완료. 시스템 설정에서 웹주소 링크 메뉴 등록/수정/순서변경/권한설정 가능
- **대시보드**: 공지사항, 국가 항공보안등급 위젯(5단계: 평시-관심-주의-경계-심각, 적용일 예약 및 변경 이력), 바로가기, 다가오는 일정
- **일정관리 캘린더** (v2.1~v2.2):
  - 시작일-종료일 기간 일정, 종일/시간 구분(시작·종료 시간)
  - 14색 고대비 팔레트, 완료 체크(칩에서 바로 토글), 🚗 차량 예약 / 🏢 회의실 예약 체크
  - 리마인더: 2주일 전 / 1주일 전 / 1일 전 / 1시간 전 (접속 중 토스트 + 브라우저 알림)
  - 담당자 카테고리: 팀원 태그(🧭박철성 · 🛡️최상일 · ✈️이은우 · 🔒이윤민) + 자유 입력, 필터
  - 보기 전환: 일 / 주 / 2주 / 월 / 년 (보기 상태 자동 저장)
  - 드래그앤드롭으로 일정 이동, ⇥ 핸들 드래그로 기간 조정
  - 기존 v2.0 일정 데이터 자동 마이그레이션
- **구글캘린더 연동** (v2.2):
  - 기존 airzetaavsec@gmail.com 2026년 일정 346건 이관 완료 (gcalId 보존 → 중복 방지)
  - SeMIS → Google: ICS 구독 주소 제공 (캘린더 🔗 버튼 → 주소 복사 → Google 캘린더 "URL로 추가")
  - Google → SeMIS: 공개 캘린더 + Google API 키 입력 시 실시간 오버레이 표시 (읽기 전용 G 칩)
- **시스템 설정** (관리자 전용): 메뉴 관리 · 사용자/암호 관리 · 데이터 관리(백업/복원/초기화/수동 동기화)

## 데이터 저장 (v2.1 — 공용 DB 실시간 공유)

- **Supabase 공용 DB** (`semis-v2` 프로젝트, 서울 리전, 테이블 `semis_store`)에
  메뉴·공지·일정·보안등급이력·사용자설정이 실시간 동기화됨 → 다중 사용자 공유
- **오프라인 폴백**: 네트워크 단절 시 localStorage로 계속 동작, 변경분은 대기열에 보관 후 재연결 시 자동 업로드
- **실시간 반영**: 다른 사용자의 변경이 Supabase Realtime으로 즉시 반영 (CDN 차단 시 30초 폴링 폴백)
- 헤더의 동기화 배지(🟢 실시간 / 🟡 동기화 중 / 🔴 오프라인)로 상태 확인, 클릭 시 수동 동기화
- 백업 파일 복원 시 복원본이 공용 DB에 우선 반영됨(forcePush)

## 초기 계정

초기 암호는 별도 전달(보안상 문서 미기재). 로그인 후 시스템 설정 → 사용자/암호에서 변경 권장.
암호 변경 내역·추가 사용자는 해시로만 저장되며 공용 DB를 통해 전 기기에 공유됨.
